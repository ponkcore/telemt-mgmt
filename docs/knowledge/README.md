# docs/knowledge/ — project knowledge base

The Architect's **Phase-0 Recon** reads everything here BEFORE designing. Drop in anything
that should shape architecture decisions:

- domain notes, glossaries, business rules
- prior-art / fork-or-build evaluations
- platform and provider constraints (rate limits, quotas, SLAs)
- house conventions not already captured in `.opencode/project.jsonc`
- links to external standards the project must honour

Keep each note focused and dated. These files are read by humans and by the Architect's
session — write them for a fresh reader with no prior context.

> This README is not validated as an artefact (filename `README.md` is skipped by
> `validate_docs.py`). Real knowledge notes can be plain markdown without frontmatter.
